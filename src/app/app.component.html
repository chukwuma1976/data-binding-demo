The value of the title variable has been added using interpolation which is one-way data binding from source-to-view. This value is being used to generate the header above. Expressions can also be interpolated. Double curly brace syntax is used like this {{valueOfCurlyBraces}}.

The above image is generated using property binding which is also one-way data binding from source-to-view. The value of currentImage is bound to the property src. Square bracket syntax used like this [dataToBind]. Property binding can also be used with class properties or style properties as well as attributes.

The below message is rendered using interpolation with and expression and the color of that message is determined using style property binding.

{{isDangerous ? 'Warning Danger!' : 'It\'s All Good!'}}

The Toggle Alert button use event binding to toggle the alert message and its color. The Toggle Image button use event binding to toggle the alert message and its color. Event binding is one-way data binding from view-to-source. The event binding syntax uses the paretheses like this (event)='someFunction()'.

The following input field uses two way data binding which is view-to-source and source-to-view. The input is subsequently displayed using interpolation. The syntax is [(ngModel)]="inputText". This is also known as 'banana in a box' syntax.

Inputted Text: {{inputText}}
